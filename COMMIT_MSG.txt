Internal CSV viewer: fix heavy dark rows in light theme

The internal viewer's odd rows used `.vgrid-row.odd { background: var(--bg-soft, #131a27); }`,
but --bg-soft is never defined anywhere, so the dark-navy fallback #131a27 was always applied
-- fine on the dark theme, too heavy on light. The totals row was hardcoded to #1d2738 (also
dark).

Added light-theme overrides in app.css:
  :root[data-theme="light"] .vgrid-row.odd    { background: #f2f5f9; }
  :root[data-theme="light"] .vgrid-row:hover  { background: #fbeecb; }
  :root[data-theme="light"] .vgrid-row.totals { background: #eaf0fb; }

(The earlier light-zebra fix only touched the standalone csv-viewer.html; this is the
internal OpenProteo viewer.)

Docs: .claude/2026-07-22-internal-viewer-light-zebra.md and CLAUDE.md.

Verify: app.css only. It is a static asset in the WAR -- rebuild the WAR and hard-refresh
(Ctrl+F5) since the CSS is browser-cached.
