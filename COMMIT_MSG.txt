Stop: forcibly abort AS400/DB2 SQL extractions by closing the connection

Stop still left a SQL extraction RUNNING (seen on a TEST run of SQL_EXTRACTION
against the AS400 jt400 source): the control wiring was correct, but jt400
Statement.cancel() does not reliably interrupt a running query/fetch.

Fix: on Stop, in addition to cancel(), close the statement AND the connection of
the running extraction (closing the socket makes the blocked call throw — the
reliable cross-driver abort).
- RunControl.aborter (Runnable); SqlSupport.exportCsv registers cancel+close-st+
  close-conn; runSql wires it. engine.stop() runs the aborter on a daemon thread
  (never blocks the Stop response) plus the existing cancel()/destroyForcibly().
- The resulting SQLException is caught (exitCode set), executeStep returns not-ok,
  and the run finalises ABORTED (loop for normal runs; testStep/runSessionStep now
  map aborted -> ABORTED instead of FAILED).

csvsql (H2) keeps cancel(). Compiles (101 classes).
