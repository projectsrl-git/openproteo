feat(vars): nested / indirect ${var} resolution (factory pattern)

Variable resolution is now iterative and innermost-first, so a variable
can build the name of another variable. With targetId=T1,
${TargetDestination.${targetId}} resolves the inner ${targetId} first
(-> T1) and then ${TargetDestination.T1} to that variable's value.

- Regex matches the innermost ${...} (a name cannot contain $ { }).
- The pass repeats up to a depth cap and stops once the string is stable,
  so a self-referential value (a -> ${a}) is left as-is with no infinite
  loop. Unknown names still resolve to the empty string.
- Single-level behavior is unchanged; this applies everywhere resolve()
  is used, including gate conditions.

Tested: factory pattern, indirection chain, unknown, self-reference,
missing inner. 76 classes compile. Documented in USAGE / README.
