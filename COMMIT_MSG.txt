Variables page: edit workflow tags (single and common feeds)

Workflow tags are now editable on the Variables page with the same name/value
criterion as the other variables.

- web/ApiController: /api/var-catalog exposes each workflow's tags as a
  comma-separated string; VarSaveReq.FeedEdit gains a tags field and
  /api/variables/save applies it (comma-separated, ${var} allowed; null leaves
  tags unchanged). Edits are validated by the parser before any file is written,
  like the other edits.
- templates/variables.html: a "tags" line in the Workflow variables section
  (single feed) and a common "tags" line in the multi-feed view, wired into the
  save payload.

Output-data definitions were already editable here: they are stored as step
params (outputData.<var> = description) and the Variables page already renders
and saves step params, per feed and across common steps. No change needed for
those.

Compiles (104 classes); variables.html passes node --check.
