Add xlsx2csv executor: Excel sheet -> CSV via POI (streaming)

New internal STEP that reads one sheet of an .xlsx workbook with the POI
streaming event API (OPCPackage + XSSFReader + SAX, constant memory), projects
the selected columns (by header name or column letter, with order and rename),
and streams the result to a CSV through the shared writer -- so its output drops
straight into a csvsql <input> (chain xlsx2csv -> xlsx2csv -> csvsql).

All cells are rendered to text deterministically (NOT as Excel displays them):
shared/inline/formula strings verbatim; boolean TRUE/FALSE; error empty;
date-styled numbers via DateUtil + a fixed dateFormat (default yyyyMMdd, UTC);
plain numbers via BigDecimal.stripTrailingZeros().toPlainString(); empty cells
preserved. Output carries no BOM.

CSV writing is refactored into a shared ds/CsvWriter (open/header/row/split/close,
UTF-8, CRLF, RFC-4180 quoting); sql, csvsql and xlsx2csv now emit byte-identical
CSV with the same ##VAR outputs (rowCount/csvFile/csvFiles/csvParts, plus
outputFile for xlsx2csv).

IMPORTANT: unlike csvsql's H2 (pure JDBC, runtime-only), Apache POI is a
COMPILE-TIME dependency. The WAR will not build until POI and all its transitives
are on Nexus -- run xlsx/PoiProbe first (see xlsx/README_POI.md). poi + poi-ooxml
5.2.5 are pinned (not BOM-managed); keep poi-ooxml-lite, not -full.
