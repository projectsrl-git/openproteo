Viewer line numbers + go-to, dequote drops blank lines, Waiting-approval column

1) Viewer (CSV / TXT / XML-JSON): row and line numbers with a "go to" box.
   - The CSV grid gained a sticky "#" gutter column with the 1-based row number and a "go to
     row" control that scrolls to the row and outlines it; the gutter width is folded into
     rowWidthPx() so header, rows and the column resizer stay aligned.
   - TXT/log files already had a line-number gutter and now get the same "go to line" box,
     highlighting the target line.
   - Formatted JSON/XML was rendered as a bare <pre> with no line numbers; it now goes through
     the same line-numbered virtual list, so it gets numbers, the line count in the meta line
     and "go to line" too. Shared helpers gotoBox() and renderLines() were added.

2) The CSV de-quoting executor copied every physical line, so the stray line breaks left at
   the end of a file (and empty lines in the middle) survived as empty records. Blank lines are
   now skipped and counted in a new output variable blankLinesRemoved (also reported in the
   summary log). A line made only of delimiters is a valid row of empty fields and is kept.

3) Operations overview: bucketFor folded WAITING_APPROVAL into "running", so runs paused on a
   MANUAL GATE could not be seen. WAITING_APPROVAL now has its own bucket: a "Waiting approval"
   tile and a "Waiting appr." by-source column, both immediately after Running and both
   clickable to drill. The "Other" tile/column is no longer displayed; it is kept as a fallback
   and reappears only if a feed really ends in an unmapped status (SKIPPED/REJECTED), so the
   per-source columns keep reconciling with Total instead of silently dropping feeds. The Mix
   bar gained a waiting segment.
