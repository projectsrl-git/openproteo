Variables page: edit step core fields (SQL query) incl. across feeds

The variables manager can now edit step CORE fields (query, source, dest,
datasource, ifsPath, csvFile, delimiter) — chiefly the SQL query — not just step
params, and apply them across multiple selected feeds for steps that share the
same step id.

- /api/var-catalog: each step also returns `fields` (non-null core fields).
- /api/variables/save: StepEdit gained `fields`; applyStepField sets the NodeDto
  field. Edits are still regenerated + validated before any write (all-or-nothing).
- variables.html: single-feed shows step core fields (query as a textarea) + params;
  multi-feed adds a "Common steps" section (step ids present in every selected feed)
  where editing a field/param applies the same value to all selected feeds; shared
  values are pre-filled, differing ones show a "(differs)" hint. Save distinguishes
  var/field/param scopes.

Compiles (103 classes). Round-trip test: a query edit persists through XML. JS verified.
