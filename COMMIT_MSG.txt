UI polish: light-theme contrast + environment header badge

Two related header/theme improvements.

1) Light-theme polish (app.css, scoped to :root[data-theme="light"]) — the light
   theme lacked contrast between field placeholders and real values because only
   .field inputs had a light placeholder colour; other inputs used the browser
   default (~text colour, dark). Now: a global lighter italic ::placeholder
   (#98a4b3, opacity 1) for all inputs/textareas; every input reads light (white
   bg/--ink/--line); real values at font-weight 500 so they stand apart from the
   italic hints; a clearer focus ring; dim/secondary text nudged for contrast. The
   dark theme is untouched.

2) Environment header badge (DEV/SIT/UAT/PROD) — distinguish installations from the
   header. AppProperties.environment (config orchestrator.environment=PROD; empty
   hides it). GET /api/env -> {environment, host}. theme.js (loaded on every page,
   already injecting the theme toggle) also fetches /api/env and injects an
   .env-badge into .topbar (single point, no per-page edits); PROD adds .is-prod.
   CSS: PROD is white-on-red and prominent with a red accent line under the topbar;
   DEV neutral, SIT blue, UAT amber; host shown on hover (covers the "obvious server
   name" alternative); light-theme variants included.
