diff executor (Batch 5): CSV_KEY key substring L/R

Implements the design's Substring L/R on key columns (section 5.1). Applies on
top of Batch 1-4 (all on main).

The design referred to reusing a grouping widget's substring control, which does
not exist in this codebase. Instead the substring is a compact per-column suffix
inside the existing keysA / keysB fields, so it is fully backward compatible
(plain names unchanged) with no model change:
- NDG          -> whole column
- NDG:L8       -> first 8 chars  -> LEFT(NDG, 8)
- CODCLI:R4    -> last 4 chars   -> RIGHT(CODCLI, 4)

This lets two feeds whose keys differ only by padding/prefix reconcile (e.g. A
stores AB1234, B stores 1234, via code:R4 <-> code).

- runDiffKey: each key column token is parsed by keyColSql into an SQL expression
  (bare column, or LEFT(col,n) / RIGHT(col,n)); these build the key CONCAT_WS.
  Invalid tokens are rejected with a clear message. Raw tokens are still echoed
  verbatim in the report's Key A / Key B lines.
- Designer: a hint under the key inputs documents the :L<n> / :R<n> syntax; keys
  stay free-text (no new widget).
