Per-feed maintenance lock (G4)

Adds a lock flag to block execution of a feed under maintenance.

- A locked feed refuses manual and scheduled (cron) runs; step-by-step TEST runs
  stay allowed so it can still be configured/verified during maintenance.
- Model WorkflowDef.locked; parser reads / writer emits root @locked (mirrors
  production); WorkflowDto.locked + toDto + designer save round-trip it.
- engine.start() refuses locked feeds (audited RUN_REJECTED) — one point that also
  covers the cron scheduler. POST /api/workflows/{feedId}/run returns 409 locked
  with a clear message. POST /api/workflows/{feedId}/lock?locked=true|false
  toggles and persists the flag (rewrite XML + reload + audit FEED_LOCKED/
  FEED_UNLOCKED). /api/overview/feeds and /active expose locked.
- UI: designer "Maintenance lock" switch; dashboard lock badge + Run disabled +
  quick Lock/Unlock per row; workflow header LOCKED badge + Run disabled;
  Operations lock glyph in drill-down and executions.

Round-trip test confirms the flag persists. Compiles (99 classes); all touched
JS verified.
