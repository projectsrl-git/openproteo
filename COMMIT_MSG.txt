Operations: per-step output data column, read from the last run

Let each step declare named run variables to surface, each with a description,
and show their values in an OUTPUT DATA column in the Operations feed list.

The values are read straight from the last run's persisted record (run.vars),
so they come from the run history with no extra capture: whatever the step put
into ${...} is already saved in _runs/{runId}.json. The last run is used even
when it failed.

- templates/designer.html: new per-step "Output data" field (one per line:
  variable = description). Stored as step params outputData.<variable> =
  description (no model/parser/writer change).
- web/ApiController (/api/overview/feeds): for each feed, read the most recent
  non-test run and resolve each declared outputData.<var> against that run's
  vars; expose an outputData list of {label, value}.
- templates/overview.html: new "Output data" column in the drill-down feed list
  rendering description: value pairs.

Compiles (104 classes); overview/designer pass node --check.
