Performance, execution-control and Operations improvements. Compiles (99 classes);

G2) Faster ResultSet -> CSV writing (output byte-identical)
- CsvWriter: skip UTF-8 byte counting unless splitting by MB (a full per-row pass
  removed in the common case); reuse one StringBuilder + one char[] across rows
  (no per-row String/StringBuilder allocation); single-pass field quoting; 512 KB
  output buffer. Round-trip self-test confirms identical bytes.
- SqlSupport: setFetchSize(1000) on the streaming extract to cut JDBC round-trips
  (extraction-side, best-effort).

G3) Adaptive, resource-gated parallel run scheduler
- Config: orchestrator.max-parallel-runs (default 1 = legacy sequential),
  orchestrator.run-admission-headroom-mb (default 256), orchestrator
  .scheduler-tick-sec (default 20).
- The single-thread executor is replaced by a cached daemon pool bounded by an
  admission step. start() and gate-resume enqueue(runId, task); a synchronized
  schedule() admits while active < max-parallel AND (active==0 OR free heap >=
  headroom), and each finishing/pausing task re-runs schedule() so a freed slot
  admits the next. A wf-scheduler tick re-checks resources every tick to admit
  runs deferred for memory. Per-feed lock unchanged (no two runs of one feed);
  test runs use their own executor and don't count against the limit.
- Operations Resources panel: JVM heap used/available, processors, load average,
  running/queued/waiting/test counts and "Parallelism (admitted / max)" with runs
  waiting for a slot (GET /api/overview/resources).
- Default max-parallel-runs=1 reproduces previous behaviour exactly; admission
  algorithm simulated standalone (peak concurrency == max for {1,3,5}, all tasks
  completed, no deadlock).

G4) Per-feed maintenance lock
- A locked feed refuses manual and scheduled runs; step-by-step TEST runs stay
  allowed so it can be configured/verified during maintenance.
- WorkflowDef.locked persisted as root @locked (parser/writer); WorkflowDto.locked
  + toDto + designer save round-trip it (verified). engine.start() refuses locked
  feeds (one point, also covers cron). POST /api/workflows/{feedId}/run returns 409
  with a clear message; POST /api/workflows/{feedId}/lock?locked=... toggles and
  persists (rewrite XML + reload + audit FEED_LOCKED/FEED_UNLOCKED).
- UI: designer "Maintenance lock" switch; dashboard lock badge + Run disabled +
  quick Lock/Unlock per row; workflow header LOCKED badge + Run disabled;
  Operations lock glyph in drill-down and executions.
