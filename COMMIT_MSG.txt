Bulk actions: add lock/unlock/clear-history; enable them on Operations drill

Extend multi-select bulk actions and bring them to the Operations feed list.

- templates/dashboard.html: the selection bulk bar gains Lock, Unlock and
  Clear history, alongside the existing Run and Delete. Each iterates the
  selected feeds over the existing per-feed endpoints (/lock?locked=true|false,
  /clear-history) via the existing runSeq helper, then reports ok/failed.
- templates/overview.html: the drill-down feed list (opened from the rollup
  totals) becomes multi-selectable -- a select-all header checkbox and per-row
  checkboxes -- with a bulk bar offering Run, Lock, Unlock, Clear history and
  Delete. The selection is kept in a small set so it survives the periodic
  (20s) feed refresh that re-renders the list. Actions reuse the same per-feed
  endpoints and refresh the view when done.

No backend change: clear-history still refuses PRODUCTION feeds and feeds with
an active run, and those are reported as skipped/failed in the summary.

Both templates pass node --check; no literal \n/\r, no unsafe Thymeleaf inline.
