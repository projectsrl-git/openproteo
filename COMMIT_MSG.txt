fix: save HTTP 400 for split/safecopy/dequote (parser exec whitelist)

Saving a workflow re-validates it by writing the generated XML and
re-parsing it with WorkflowXmlParser. The parser's allowed-exec
whitelist - and the "internal" set that decides whether a script
attribute is required - had not been updated for the recently added
executors split, safecopy and dequote. Parsing a workflow that used
exec="safecopy" therefore threw, and save returned HTTP 400 (the
corporate proxy then replaced the error body with its own HTML page,
hence the misleading "intercepted by a proxy" message in the client).

Added split, safecopy and dequote to both the exec whitelist and the
internal-executor set in WorkflowXmlParser. The engine and the designer
already supported these execs; only the save-time parser was stale.

Verified: a workflow with safecopy (pattern "*.md5, *.tar"), dequote and
split now parses successfully. 76 classes compile.
