fix(safecopy): support multiple comma-separated wildcard patterns

safecopy received the wildcard field as a single glob, so a value like
"*.md5, *.tar" only matched one pattern at run time. The pattern is now
split on comma/semicolon and each glob is streamed, with results
de-duplicated by file name. Designer placeholder/hint and USAGE updated.
Tested: "*.md5, *.tar" copies the .md5 and .tar files and skips others.

Note (no code change): the "HTTP 400 as text/html / Bad Request" seen on
Validate & save is not an OpenProteo validation error - save() posts JSON
and the client flags any non-JSON reply. A terse HTML 400 is the
corporate proxy/WAF rejecting the POST body (or an expired SSO session).
Isolate by re-logging in and by saving with individual fields/steps
removed to find the token the filter blocks; capture the blocked-request
id from the browser Network tab.
