diff executor (Batch 3): TEXT line-diff mode

Third and final comparison mode of the diff / reconciliation design
(.claude/DIFF_EXECUTOR.md). Applies on top of Batch 1+2 (already on main).

- runDiff dispatches TEXT to runDiffText (line-oriented). Small files (both
  <= textMaxLines, default 2000) get a real LCS diff (bottom-up DP + forward
  walk) reporting only_in_A / only_in_B lines with their line numbers. Larger
  files fall back to a streaming positional line comparison (line_changed, plus
  surplus as only_in_A / only_in_B), with the degradation noted in the report --
  TEXT is intended for small control files, CSV_POSITIONAL / CSV_KEY stay the
  scalable paths. (Resolves the open cap/fallback question in the design.)
- Same report pair: <name>_recon_report.md (headline, config incl. fallback note,
  summary lines A/B/common, totals with % over the larger file) and
  <name>_recon_differences.csv (line, label, valueA, valueB, category). Output
  vars: diffResult, diffCount, linesA, linesB, commonLines, onlyInA, onlyInB
  (+ changedLines in fallback), reportFile, differencesFile.
- Designer: TEXT added to the mode dropdown; when selected, a Max-lines field and
  a note that the delimiter is ignored. Config stays params-only: no model, DTO
  or writer change.

Docs: .claude/2026-07-14-diff-executor-batch3-text.md and CLAUDE.md.

Verify: the actual runDiffText source was executed standalone on sample files --
LCS path (common 4, only_in_A 1, only_in_B 2, 33.33% over the larger file) and
positional fallback (textMaxLines=3 -> common 3, changed 2, only_in_B 1, 50%),
with failOnDifferences flipping the exit code. designer.html passes node --check;
no literal \n/\r; no unsafe Thymeleaf. Full Maven build to be confirmed on deploy.
