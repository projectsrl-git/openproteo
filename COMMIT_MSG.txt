Validate: add businessDateNotBefore check (min date or today)

New CSV-vs-DataSchema check "businessDateNotBefore": the business-date value must
not be earlier than a minimum date. The minimum is the businessDateMin
param/variable when set, otherwise today. Boundary inclusive: a value equal to the
minimum passes; only earlier values fail.

- engine/InternalSteps.java (validate): parse the business-date column with the
  configured dateFormat and compare each value against the minimum; count and list
  the offending rows. SKIPs when there is no dateFormat or no business-date column;
  FAILs with a clear reason on an invalid businessDateMin. Added to the default
  checklist.
- templates/designer.html: new "Business date min (var/date)" field on the validate
  node (placeholder ${businessDateMin}, empty = today).
- workflows/SAMPLE-06-validate.xml: businessDateMin param wired to ${businessDateMin}.

The business-date column already defaults to the recordBusinessDate variable
(previous commit), so this builds on top of it. Compiles (103 classes); date
parse/compare verified.
