diff: TEXT_SET mode, CSV_KEY match aggregates, attributes-checked total

Three additions to the diff executor.

1) New mode TEXT_SET: line membership, order-independent. Treats each file as a
   set of lines and reports lines only in A (A->B: in A, not in B) and only in B
   (B->A). Uses hash sets (O(n+m), scales), unlike the ordered LCS TEXT mode. One
   differences file carries both directions (category only_in_A / only_in_B).

2) CSV_KEY match aggregates: each match gains a per-match agg
   (value | sum | count | count_distinct; default value = current behaviour). For
   a key group the aggregate is computed and compared A vs B: sum over the numeric
   value (empty cells ignored; compared with BigDecimal), count of rows, or count
   of distinct values. The multi-occurrence inconsistent_key check is bypassed for
   aggregates and the comparison is numeric. Designer gains an Aggregate dropdown
   per match; the report echoes agg per match. H2 2.1.214 has no TRY_CAST, so sum
   requires numeric columns (a non-numeric value surfaces a clear H2 error).

3) Summary total attributes checked: POSITIONAL and CSV_KEY reports now show, and
   expose as attributesChecked, the sum over both files of
   (checked attributes x rows of that file) = attributesCompared x (rowsA + rowsB),
   and also print rows in A / rows in B. Not applicable to TEXT modes.

Docs: .claude/2026-07-15-diff-textset-aggregates-checked.md and CLAUDE.md.
