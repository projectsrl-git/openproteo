Add csvsql executor: H2 SQL/joins across CSVs -> CSV

New internal STEP that runs an arbitrary SQL query (joins, aggregates,
subqueries, CTEs, window functions) across several CSV files and streams the
result to a new CSV, like the other CSV-producing steps. The user writes only
the SELECT over table aliases; all H2 plumbing (CSVREAD staging, the export) is
generated and hidden.

- Pure java.sql (no org.h2.* imports): the project compiles and the app starts
  with no H2 on the classpath; H2 is a runtime-only dependency. A csvsql step
  fails with a clear message until the driver is present (see h2/README_H2.md).
- Everything stays VARCHAR (CSVREAD); the user casts inside the query for math.
  String equi-joins keep leading zeros; fixed-width yyyyMMdd dates compare as
  strings.
- Temporary file-mode H2 DB under ${stepDir}, deleted in finally. The user
  query runs verbatim via Statement.executeQuery and is streamed through the
  shared CSV exporter (UTF-8, CRLF, header, row/MB split, ##VAR rowCount/
  csvFile/csvFiles/csvParts), so it is fully LOOP-compatible. Output is written
  without a BOM so csvsql->csvsql chains are safe.
