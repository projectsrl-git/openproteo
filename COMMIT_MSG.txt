Fix csvsql: auto-detect input delimiter (was forced to output)

csvsql read every input with the step's OUTPUT delimiter (default ';'). A
comma-separated input was therefore parsed as a single column named after the
whole header, so SELECT * "worked" but WHERE <col> failed with
  Column "<col>" not found [42122]

Fix: decouple input and output separators and auto-detect the input one.
- New optional per-input <input ... delimiter="">; blank = auto-detect.
- InternalSteps.detectDelim() sniffs the header (comma/semicolon/tab/pipe
  outside quotes, ignores BOM, falls back to the output delimiter); each input
  is staged with its own separator and the log records which was used.
- ApiController csvsql preview reuses the same detection.
- model/parser/writer/DTO carry the per-input delimiter; designer adds a small
  "Sep" field (placeholder "auto") plus a hint; XML emits delimiter only when set.

A comma EORFULL extract now parses into real columns (CODCLI, NDG, ...), so
WHERE NDG = '...' works; the output delimiter stays independent.

Compiles (96 classes). Confirm on UBS (live H2 not runnable in the sandbox).
