Workflow-level OUTPUT DATA (1.5)

Output data could only be defined per step (step params outputData.*). It can now
also be defined at the workflow level, with the same "variable = description"
textbox on the Variables page.

- Model: WorkflowDef gains Map<String,String> outputData (var -> description).
- XML round-trip: a <outputData><var name=".." desc=".."/></outputData> section
  (mirrors <variables>); parser reads it, writer writes it from
  WorkflowDto.outputData (populated in toDto). DOM round-trip verified standalone.
- Save + display: the Variables page gets a "workflow output data" textarea (one
  var = description per line) at the workflow level, in both the single-feed and
  the multi-feed common editor (differ-across-feeds aware, like tags). var-catalog
  exposes it as text; VarSaveReq.FeedEdit (and FeedImport) gain outputData;
  applyEditsToDto parses the text into dto.outputData (full replace), exactly like
  the step-level output-data parsing. /api/overview/feeds folds the workflow-level
  definitions into the output-data variable set, so they feed the last-run and
  all-runs output-data display.
