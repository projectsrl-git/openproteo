Variables matrix: spreadsheet editor with feeds on rows and variables on columns

New page /matrix (matrix.html) to insert and edit everything in a single screen, like a
spreadsheet -- complementing the existing Variables page.

Layout: rows are feeds (sticky first column with feedId + name; PROD feeds get a red left
border), columns are the union of every workflow variable name across all feeds, plus two
optional meta columns (tags as text, PROD as a checkbox). The header row and the feed column
are sticky and the sheet scrolls both ways.

Editing: type in any cell; only the cells you touch are sent. Edits live in an EDITS map keyed
by feedId+kind+name so filtering never loses them, and dirty cells are highlighted. An empty
cell means the variable is not defined for that feed (placeholder "not set") and typing a
value CREATES it on save. "+ Add column" introduces a new variable name across all feeds; the
header's fill button copies a value down to every visible feed; Arrow Up/Down and Enter move
between cells; pasting a TSV block from Excel fills cells to the right and below. Filters for
feeds and for column names, plus "only columns that differ" to show just the variables whose
value is not identical across the visible feeds. Discarding asks for confirmation and leaving
with unsaved edits warns.

Backend: no new save endpoint -- the grid posts the existing POST /api/variables/save with
{edits:[{feedId, vars:[{name,value}], tags?, production?}]}. var-catalog now also returns
`production` per workflow (it did not before, so the Variables page PROD hint was always
"non-production" as well). Route added in PageController, links from the dashboard and the
Variables page.

Docs: .claude/2026-07-23-variables-matrix-sheet.md and CLAUDE.md.

Verify: matrix.html passes node --check with zero literal \n/\r and no unsafe Thymeleaf; the
save-payload grouping was executed in Node using the real source (vars, tags and production
grouped per feed, newly created variable included). Java not compiled in the chat sandbox --
run the Maven build on deploy.
