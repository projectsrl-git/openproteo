Variables page: mass-edit "Step mode" (skip / on hold) with the editor's dropdown

The Variables (mass-edit) page could edit step fields, params, output data and on-success
delete across the selected feeds, but not the step mode. Added a "step mode" dropdown per
step -- same options as the designer step editor (leave unchanged / normal /
skip (passthrough) / on hold (pause)) -- so skip or pause can be set massively on the common
steps of many feeds, using the same dropdown found in edit.

- variables.html: valSelect "step mode" in both the single-feed step view and the
  common-steps (multi-feed) view; the hint shows the current mode ("currently: ..." or
  "differ across feeds"). The collector adds scope 'stepMode' to the step edit.
- ApiController: StepEdit gains a stepMode field; applyEditsToDto maps it to the step DTO
  exactly like the designer's setStepMode (skip -> skip=true/onHold=null; onHold ->
  onHold=true/skip=null; normal -> both cleared). var-catalog now also returns skip/onHold
  per step so the "currently" hint is accurate.

Docs: .claude/2026-07-22-variables-step-mode-massedit.md and CLAUDE.md.
