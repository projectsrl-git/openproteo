Variables page: edit output-data and on-success-delete per step

Make the orchestrator-internal step settings editable from the Variables page
with dedicated controls (mirroring the designer), instead of raw param lines.

- templates/variables.html: per step (single feed and Common steps) show an
  "output data" textarea (one per line: variable = description), an
  "on-success delete (path)" field and an "on-success delete (type)" field.
  The raw deleteOnSuccess / deleteOnSuccessType / outputData.* params are hidden
  from the generic param list to avoid duplication; Common steps use the usual
  "all equal, else type to set for all" behaviour.
- web/ApiController (/api/variables/save): StepEdit gains outputData,
  deleteOnSuccess and deleteOnSuccessType (null = unchanged). outputData does a
  full replace of the outputData.* params; the other two set/append, or remove
  when blank. New setOrRemoveParam() helper. All edits keep going through the
  parser validation before any file is written.

Compiles (104 classes); variables.html passes node --check.
