Add read-only Cert Identity Probe diagnostic endpoint (GET /diag/whoami)

Temporary diagnostic to determine how the UBS personal browser-certificate
identity reaches OpenProteo (TLS client cert at Tomcat vs identity headers
injected by an upstream proxy), to design the real authentication filter.

- Adds com.legalarchive.orchestrator.diag.CertIdentityProbeController (package
  adjusted from the handoff's example com.ubs.openproteo to this project's base
  com.legalarchive.orchestrator; auto-detected under the @SpringBootApplication
  scan root). Renders one HTML page with: (1) the javax.servlet.request
  .X509Certificate chain, (2) all request headers (SSL_CLIENT_* / verify / UBS
  identity headers), (3) connection facts (isSecure, scheme, remoteAddr, server
  name, URL, cipher). No new dependencies, no pom change, no JavaScript.
- Handoff doc placed at .claude/CERT_IDENTITY_PROBE_INTEGRATION.md.

Open it through the same HTTPS/SSO entry point users use, with the cert-holding
browser: https://<host>/openproteo/diag/whoami. REMOVE or lock down after the
identity source is confirmed (it exposes all request headers).

Compiles (100 classes).
