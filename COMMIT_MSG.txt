feat(loop): 1-based loopIndex + configurable zero-padded loopIndexString

- ${indexVar} (default loopIndex) is now 1-based (was 0-based).
- New ${indexStringVar} (default loopIndexString): the 1-based index
  left-padded with '0' to a configurable width N (indexPad, default 3),
  e.g. 001 or 00005 - convenient for ordered output file names.

Implementation:
- LoopDef: + indexStringVar, + int indexPad.
- WorkflowEngine.setLoopIndexVars(): indexVar = internal0Based + 1;
  indexStringVar = String.format("%0Nd", ...). The internal loop counter
  stays 0-based to index the item list.
- Parser / writer / NodeDto / /definition: indexStringVar + indexPad
  round-trip.
- Designer: Index var marked 1-based; new Index string var, Pad width and
  Count var fields; defaults in Add loop; preview XML; help shows a live
  padded sample.

Tested: writer/parser round-trip (idxStr, pad 5); setLoopIndexVars
0 -> 1/00001 and 4 -> 5/00005. 76 classes compile; designer JS OK.

Note: workflows that relied on a 0-based index will now see 1-based
values (intended). Docs (USAGE.md/README) updated.
