Clear history: allow PROD behind a checkbox + keep-last option (2.1/2.2)

2.1 The per-feed clear-history endpoint gains confirmProduction. A PRODUCTION feed
    is no longer hard-refused; it is cleared only when confirmProduction=true,
    otherwise it is skipped with a clear message. opConfirm is extended to support
    opts.checks = [{id,label,required,checked}]: it renders checkboxes and keeps
    the OK button disabled until every required box is ticked, passing the checked
    state to onYes. Operations drillBulkClear (switched from native confirm to
    opConfirm) counts the PROD feeds in the selection and shows a required
    "production included" checkbox when any; the dashboard offers an optional
    "also clear PRODUCTION feeds" gate. Both pass confirmProduction to the endpoint.

2.2 clear-history gains keepLast: when true it deletes every run except the most
    recent (store.delete per run) instead of the full clearOneFeed wipe, preserving
    the latest run's state, logs and working output (audited
    FEED_HISTORY_CLEARED_KEEP_LAST). Both confirm dialogs offer a "keep the most
    recent run" checkbox.

Docs: .claude/2026-07-15-clear-history-prod-keeplast.md and CLAUDE.md.

Verify: modal.js, dashboard.html and overview.html pass node --check (no literal
\n/\r, no unsafe Thymeleaf). The opConfirm required-checkbox gating was exercised
in Node (OK disabled until the required PROD box is ticked; enabled when no PROD is
present). Backend mirrors existing store.list/store.delete/badRequest/audit
patterns. Full Maven build to be confirmed on deploy.
