feat: ${sharedDir} variable so a step can create shared files

A step can now write an app-level shared file by targeting ${sharedDir}
(the orchestrator.shared-dir directory, the same folder managed by the
Shared files page). The variable is seeded in the run vars from
AssetStore.sharedDir() and is overridable by a workflow variable of the
same name.

Examples:
- sql export:  csvFile = ${sharedDir}/report.csv
- split:       output base under ${sharedDir}
- filecopy:    dest = ${sharedDir}

The written file then appears on the Shared files page and is available
to every workflow. Added to the designer cheat sheet + path suggestions
and documented in USAGE.md / README.

into the configured shared directory.
