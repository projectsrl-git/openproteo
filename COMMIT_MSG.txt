docs: convenzione deploy auto-commit + COMMIT_MSG nello zip

Da ora COMMIT_MSG.txt viaggia dentro lo zip (openproteo/), versionato,
così ogni pacchetto porta il proprio messaggio di commit.

Il deploy_openproteo.bat (consegnato a parte, fuori dal repo) e' stato
aggiornato per:
- robocopy senza /XF COMMIT_MSG.txt (il messaggio arriva dallo zip);
- dopo una build OK: git add -A, e se ci sono modifiche
  git commit -F COMMIT_MSG.txt + git push (best-effort, non blocca il
  deploy locale; nessuna modifica -> nessun commit).

Qui nel repo aggiorno solo la documentazione della convenzione
(CLAUDE.md + .claude/2026-06-16-deploy-autocommit.md) e includo
COMMIT_MSG.txt. Il codice runtime e' invariato dal turno del loop.
