Fix diff cross-workflow picker: File A/B not set on selection

Selecting File A / File B from the "...or from a workflow" dropdowns sometimes
left the File A / File B validation still complaining that the boxes are required.

Two fragilities in the designer picker:
- diffFileChosen returned early if the File input element was not found at that
  instant (if (!fsel || !fld) return), so setNodeParam was never called and the
  fileA/fileB param stayed empty.
- the absolute path was rebuilt from a data-dir attribute stored on the file
  <select>; any editor re-render (e.g. adding a match) rebuilds that select and
  drops data-dir, so a later selection could resolve to a wrong or relative path.

Fix (designer only):
- diffWfChosen bakes the absolute path (dir + path) straight into each file
  option's value, so selection no longer depends on data-dir surviving.
- diffFileChosen guards only on the select, reads the already-absolute value, and
  calls setNodeParam unconditionally (updating the input only if present), so
  picking a file always sets fileA/fileB.

Picking from a workflow fills the File A/B box with the resolved absolute path by
design (the box shows what will be compared); the user types nothing.

Docs: .claude/2026-07-15-diff-crossworkflow-picker-fix.md and CLAUDE.md.

Verify: the real diffFileChosen / setNodeParam / nodeParam were executed with a
mock DOM, including the worst case where the File input element is absent -- the
param is still set and validation passes. designer.html passes node --check; no
literal \n/\r; no unsafe Thymeleaf.
