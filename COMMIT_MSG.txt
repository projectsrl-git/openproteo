Fix Stop on SQL runs (left RUNNING); selectable home-list variables

1) Stop left a run RUNNING during SQL extraction: the DB2 sql executor never
   registered its JDBC Statement in RunControl, so engine.stop() couldn't cancel
   it. SqlSupport.exportCsv now reports the live Statement via a callback (+ fetch
   size); runSql registers it (st -> control.statement = st). Stop cancels it, the
   query aborts, the step returns not-ok and the run finalises ABORTED (loop catch
   also guarantees finalisation). Re-applied the G2 CsvWriter optimisation +
   setFetchSize lost in an earlier packaging (byte-identical, re-verified).

2) Selectable variables in the home feed list: new orchestrator.home-list-vars
   (comma-separated workflow variable names). The dashboard shows a column per
   name with each feed's value and includes those values in the row search, so
   feeds can be searched by e.g. Business Record Date. Empty = unchanged layout.
   AppProperties.homeListVars/homeListVarNames(); PageController resolves
   row.varValues + row.varSearch; dashboard.html renders dynamic columns/colspans.

Compiles (99 classes); CsvWriter byte-identity test passes; dashboard JS verified.
