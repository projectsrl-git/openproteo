Operations inline detail + CSV; Files share link + absolute path

Operations page (templates/overview.html):
- the per-feed detail is always visible; the rollup tiles now filter it instead
  of toggling a hidden panel ("Show all" resets the filter).
- inline feed filter (id/name/source/target/status) plus CSV/Copy buttons live
  in the static panel header, so the 20s auto-refresh never steals focus or
  clears typed text; the visible list is recorded as drillDisplayed.
- CSV export of the displayed (filtered) rows and of the selected (checkbox)
  rows -- two separate downloads -- and Copy-as-CSV to the clipboard. CSV is
  built client-side with String.fromCharCode line ends (no literal newline
  escapes) and rows.push([...]) (no [[ for Thymeleaf).

Files panel (static/js/filespanel.js, shared by workflow/shared/pools):
- new Share button copies an absolute direct-download link (new URL against the
  page URL); Copy path now copies the absolute path (scopeDir from the list
  response's dir + relative) so files can be referenced across feeds.
