Step-by-step testing off the main queue

Adds a dedicated test executor (wf-test-runner) and
WorkflowEngine.testStep(feedId, stepId, user) so a single step can be test-run
immediately, off the single-threaded main queue, concurrently with other feeds.

- testStep() reuses buildRun() (full var seeding) with a distinct
  <feedId>_test_<ts> runId, marks other steps SKIPPED, registers the run in
  activeRuns + controls and persists it, then runs executeStep() on the test
  pool and finalizes via finish(). It writes to the feed's normal step folders,
  so outputs persist and step N can read step N-1's output (true step-by-step).
  It does not take the per-feed lock; it is refused only if the SAME feed has a
  normal run active.
- finish() now releases the per-feed lock conditionally
  (runningFeeds.remove(feedId, runId)) so a finishing test run can never clear a
  normal run's lock.
- API POST /api/workflows/{feedId}/test-step/{stepId} -> {ok, runId, url}.
- Designer: a "Test" button on each STEP node opens the standard run page (live
  console + Stop) in a new tab; uses the SAVED workflow.

Compiles (97 classes). designer JS verified; live behaviour is UBS-side.
