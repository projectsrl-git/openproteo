diff CSV_KEY: substring L/R on match columns

Batch 5 added the COL:L<n> / COL:R<n> substring only on key columns. This extends
the exact same syntax to match columns (A and B sides), so an attribute can be
compared on a slice -- e.g. an initial (firstName:L1) or the last digits of an
account (iban:R4).

- runDiffKey: each match column token now goes through the existing keyColSql
  (bare column, or LEFT(col,n) / RIGHT(col,n)), like key columns, and the
  resulting expressions build the match CONCAT_WS. Invalid tokens are rejected
  with the same message. The label default still uses the raw tokens
  (e.g. firstName:L1+lastName) so the report stays audit-readable.
- For numeric matches the substring is applied on the text first, then compared
  numerically (BigDecimal), e.g. RIGHT(iban,4) compared as a number.
- Designer: the matches hint documents the :L<n> / :R<n> suffix on match columns
  too; fields stay free-text, no new widget.

Docs: .claude/2026-07-15-diff-csvkey-match-substring.md and CLAUDE.md.
