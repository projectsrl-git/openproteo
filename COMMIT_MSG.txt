Bulk "schema only" mode: restore dataschema/displayschema

Adds a schema-only path to POST /api/workflows/bulk (schemaOnly=true) to
re-create or refresh the dataschema.json / displayschema.json of EXISTING feeds
in bulk, without a template and without touching any workflow XML. Needed to
recover the ~140 feeds whose schema uploads were deleted by the earlier
clear-history bug.

- BulkWorkflowGenerator.parseSchemas() reads only feedId + dataschema +
  displayschema columns (no template, no XML).
- The endpoint writes the validated JSON into ${feedDir}/dataschema.json and
  displayschema.json for each row whose feed already exists (same file names and
  write as the original bulk-create), reporting per-row updated/skipped/error and
  counts. 'template' is now optional (required only for the create path).
- bulk.html: a Mode selector (create/update workflows | schema only); in
  schema-only the template is disabled with a hint, and 'updated' shows green.

Use: Bulk page -> Mode = schema only -> paste the feeds CSV (feedId, dataschema,
displayschema) -> run.

Compiles (96 classes). bulk JS verified; live write is UBS-side.
