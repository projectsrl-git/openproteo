Tag badges in Operations overview + new runtime currentDate/currentTs variables

#1 The overview feed list showed tags as a plain "tags: a, b, c" line. overview.html now
renders each resolved tag as a small pill (.tagb inside .tag-badges); app.css styles the
badges with a light-theme override.

#2 Added runtime variables currentDate and currentTs. Unlike runDate/runTs (fixed at run
start), they are evaluated NOW and refreshed before every step execution in
WorkflowEngine.loop() (yyyyMMdd and yyyyMMdd_HHmmss). So when an ON-HOLD step is resumed days
later, that step and its successors can use ${currentDate} to get today's date instead of the
original ${runDate}. feedVars and the overview tag-resolution map also expose
currentDate/currentTs (design-time value = now) so ${currentDate} resolves in previews and in
feed tags.

Docs: .claude/2026-07-22-tag-badges-and-currentDate.md and CLAUDE.md.

after deploy.
